ZX Spectrum BASIC Studio

Ready. Line 10. Adventure!

This is a friendly workshop for writing Sinclair BASIC on a modern keyboard, checking the listing, and turning it into a genuine tokenised Spectrum tape. Start small, press buttons, make the border flash alarming colours—your browser keeps a recovery copy while you experiment.

1 Your first runnable program

  1. Give the project a memorable name at the top of the page.
  2. Choose Example, or replace the listing with three lines of your own.
  3. Every stored instruction begins with a line number from 1 to 9999. Leaving gaps of ten makes later editing easier.
  4. Choose Validate. Fix red errors; read yellow warnings and decide whether they matter.
  5. Set the TAP name and an autostart line—normally your first line.
  6. Choose Download TAP, load it in an emulator, and enjoy the sight of your browser-written BASIC running as Spectrum code.
10 BORDER 1: PAPER 0: INK 7: CLS
20 PRINT AT 10,7;"HELLO SPECTRUM!"
30 BEEP .1,10
Fastest test: save the project to the cloud, publish its TAP, then choose Try in QAOP. No cassette leads, no rewinding, no mysterious squeal required.

2 The editor and validator

The large left-hand area is an ordinary text editor for numbered Spectrum BASIC. The margin shows physical text rows; the numbers you type inside the listing are the actual BASIC line numbers.

Example

Loads a small known-good program. It is an excellent starting point for fearless tinkering.

Clear

Empties the listing. Undo can bring it back, and an empty editor is required before importing BAS.

Validate

Checks line numbers, duplicates, strings, brackets, common statement mistakes, jump targets, and Spectrum variable rules.

Tokenised view

Shows the bytes the Spectrum will store. Keywords such as PRINT occupy one token byte instead of five letters.

Click a reported problem to jump to its physical source row. Validation catches many useful mistakes, but it is not a complete Spectrum emulator: a program can be structurally sensible and still do something wonderfully unintended when run.

A warning is not always an error. A jump to a missing line can be intentional on a Spectrum, which continues at the next available line. The Studio points it out instead of pretending to know your plan.

3 Saving, recovery, Undo and Redo

  • Save Project downloads an editable JSON project containing the listing, name and TAP settings.
  • Load Project opens one of those JSON projects later.
  • New Project starts afresh and asks before replacing unsaved work.
  • Undo and Redo step through source changes. A complete renumber or variable rename is one undoable action.
  • The latest project is automatically recovered from this browser after a refresh or accidental closure.
  • If work has changed since the last explicit save, the browser also warns before leaving.
Recovery is a safety net, not an archive. Browser storage belongs to this browser and device and may be cleared. Download JSON projects—or use cloud saves—for work you care about.

4 JSON, BAS and TAP: three different jobs

Project JSON

The Studio's editable master file. Use this when you want to return with the project name and export settings intact.

BAS

A readable, plain-text numbered listing. It is convenient for source archives, version control and moving listings between compatible tools.

TAP

A virtual Spectrum tape containing the real tokenised program. Use this for emulators and suitable real-hardware loading solutions.

Copy Listing

Copies the readable source to the clipboard for a forum post, message, document or another editor.

To protect the current listing, Import BAS works only while the editor is empty. Choose Clear first, then import. BAS files are used in several slightly different ways by Spectrum tools; this preview imports and exports the straightforward plain-text form.

The Autostart line tells the Spectrum which line to run after loading. Leave it blank or invalid if you want a program that loads without automatically running.

5 Renumber and Variable Explorer

Renumber without crossed wires

Open Renumber below Export. Choose the whole program or an inclusive existing-line range, then set the first new line and increment. Before changing a single character, the Studio calculates the complete result.

If a proposed number collides with an untouched line, duplicates another result, or exceeds 9999, the operation is cancelled. Literal GO TO, GO SUB, RESTORE, RUN and LIST targets follow lines that move—even when the reference sits outside the selected range. Calculated targets, strings and REM comments are left alone.

Know your variables

Variable Explorer updates as you type and separates ordinary numbers, strings, arrays and FOR/NEXT controls. Enter a new name beside a variable and choose Rename. Exact code references change; matching words inside strings and comments do not.

  • Ordinary numeric names may be descriptive, such as score or lives2.
  • Simple string names use one letter plus $, such as n$.
  • Array names and FOR/NEXT control variables use a single letter.
  • A rename is refused if the new name is invalid or already belongs to another variable.

6 Getting to grips with Spectrum BASIC

The folded Spectrum BASIC help panel is a quick memory-jogger. Search for PRINT, FOR, POKE, UDG or another topic. For a proper course and reference book, open the excellent online manual below.

Read Chapter 8: The +3 BASIC Programmer's Guide ↗

Chapter 8 begins with simple programming concepts and builds through decisions, loops, subroutines, DATA, strings, functions, arrays, colour, graphics, sound, memory and machine code. The +3 has some later facilities, but the core Sinclair BASIC teaching is extremely useful for classic Spectrum programs.

Make a decision

IF lives=0 THEN GO TO 500

Repeat something

FOR n=1 TO 10: PRINT n: NEXT n

Call a routine

GO SUB 1000, then finish it with RETURN.

Use Spectrum colour

INK is foreground, PAPER is background, and BRIGHT affects the complete 8×8 attribute cell.

Binary graphics data: BIN 11111111 means 255, not eleven million. BIN accepts only 0 and 1 and is ideal for writing readable 8-pixel UDG rows.
48K or 128K? Leave BASIC target on Auto-detect for most programs. PLAY and SPECTRUM select 128K token rules. In 128 BASIC, bytes used by 48K UDG T and U become those two commands, so UDGs are limited to A–S.

7 Cloud projects, calmly explained

Cloud storage lets the same Google account reach private Studio projects from another browser or device. Google confirms who you are; the Studio never receives your Google password.

  1. Choose Cloud Projects and sign in with Google.
  2. Enter a cloud project name and choose Save as New. This creates a private copy.
  3. After editing, choose Update Current—or Save on that project's card—to replace the current cloud copy.
  4. Choose Publish TAP only when you want public, shareable links.
  5. Use Copy Project Link for a page where someone can inspect and open their own editable copy.
  6. Use Copy TAP Link for the tape file itself, or Try in QAOP to run it immediately.

Private, published and deleted are different

  • Private: only your signed-in account can open the cloud project.
  • Published: anyone with the unlisted project or TAP link can access that published material.
  • Unpublish: removes the public links but keeps your private cloud project.
  • Delete: permanently removes that cloud project and its published TAP. Downloaded copies are unaffected.
Save before publishing. Publishing first updates the current cloud project, validates the BASIC, builds a fresh TAP and gives it a new revision URL so emulator caches do not keep an older program.

8 What is QAOP, and what happens when I click it?

QAOP is a ZX Spectrum emulator that runs in a web browser. An emulator recreates the Spectrum in software: its processor, memory, display, keyboard and tape loading behaviour.

When you choose Try in QAOP, the Studio sends QAOP the public URL of your published TAP. QAOP downloads that tape from this site, loads it into its virtual Spectrum and runs the program. The TAP remains hosted here; QAOP is the machine playing it.

  1. Save the latest cloud project.
  2. Publish—or Update TAP if it was already published.
  3. Choose Try in QAOP on the project card.
  4. Allow the emulator a moment to load. Click or tap its screen if the browser is waiting for interaction.
  5. Use the emulated Spectrum keyboard controls as required by your program.
Sharing trick: the direct TAP URL can also be passed to other compatible online emulators or downloaded for desktop emulators and hardware workflows.

9 BASIC and user-defined graphics

On a real Spectrum, UDGs are entered in Graphics mode. In this editor, write \A 

10 PRINT AT 10,15;"\A"
20 PRINT AT 12,11;"UDGS B TO U: \B\C\D ... \U"

The readable backslash notation remains unchanged in JSON and BAS source. Only the tokenised TAP receives the special UDG bytes, ready to display whatever patterns are currently installed in the Spectrum's UDG definitions.

Use the UDG Designer in the top navigation to draw graphics, store their colours, build screens and export its graphics TAP. A future Studio workflow can bring graphics data and BASIC source closer together; for now they remain separate projects so each tool stays predictable.

10 If the Spectrum gremlins arrive

  • TAP will not export: open Problems and fix every red validation error.
  • Old program appears in QAOP: Update TAP again and launch using the newly revised link.
  • BAS will not import: Clear the editor first. Binary/tokenised BAS variants are not accepted by this text importer.
  • Renumber refuses: read the collision message and choose a first line or increment that leaves untouched lines free.
  • Cloud buttons are missing: sign in, open the cloud project, and ensure it is the current project before publishing.
  • You refreshed: relax—the latest browser recovery copy should return automatically.

Write ten lines. Run them. Smile. Then add ten more. That is how Spectrum epics begin.
