PLAYER'S GUIDE

ZX Spectrum UDG Graphics Editor

Make something wonderfully Speccy

Welcome! This editor turns tiny 8×8 pictures into reusable Spectrum graphics, complete screens, BASIC data and a ready-to-load TAP file. You do not need to be a machine-code wizard. Start with one tile, experiment freely, and let the editor handle the fiddly bytes.

Created with love for the Spectrum scene by Keilan Knight.

1 Your first screen in ten minutes

  1. Name the project. Something grand is encouraged.
  2. Choose Bank 1 and UDG A. This is your first 8×8 character.
  3. Draw on the grid. Try a brick, star, key, alien or tiny teapot.
  4. Choose its INK, PAPER and BRIGHT. Check how it repeats in the 6×6 preview.
  5. Make a few more UDGs. B could be a floor; C could be treasure.
  6. Move to the Screen Editor. Pick a UDG and paint it onto the 32×24 screen.
  7. Save Project. This downloads your editable project file.
  8. Download TAP. Load it in an emulator or onto suitable real-hardware storage.
No panic button required: your current work is also kept automatically in this browser. If the page refreshes, the latest recovery copy comes straight back.

2 Projects, saving and recovery

New Project

Starts with four empty banks and one blank screen. If you have unsaved changes, you can cancel before anything is cleared.

Save Project

Downloads a JSON project containing every UDG, colour, bank, screen and editor setting. Keep this file if you want a portable, long-term copy.

Load Project

Opens a previously saved project file and restores the whole workshop, not just the currently visible screen.

Browser recovery

Your latest work is stored locally and restored after a refresh or return visit. The browser also warns before leaving with changes not saved to a project file.

Browser recovery belongs to this browser and this website address. Clearing site data, changing device, or using private browsing may remove it—so a downloaded project file is still your trusty cassette-box backup.

3 UDGs: tiny pictures with big jobs

A Spectrum UDG is an 8×8 user-defined graphic. Each row becomes one byte, but you can forget the arithmetic: click pixels and the editor calculates it. The familiar UDG letters are A–U, giving 21 slots per bank.

The four banks

Each bank has its own A–U, giving up to 84 designs. Bank 1 is a good home for common scenery and interface pieces. Use Banks 2–4 for another level, animation frames, enemies, alternate themes, or special screens. A designed screen can mix tiles from all four banks.

Duplicate Bank copies a complete bank—including every UDG's saved colours—into the next unused bank. Banks already used by a screen are protected and are not considered free.

Drawing the 8×8 grid

  • Mouse: left-drag draws; right-drag erases.
  • Touch or pen: tap an empty pixel to add it; tap a filled pixel to remove it.
  • Clear: empties the selected UDG. Invert: swaps every on/off pixel.
  • Duplicate: copies the design and its saved colours into the next empty UDG slot.
  • Copy / Paste UDG: copies pixels and saved colours, ready to paste over any selected letter in any bank.
  • Mirror, rotate and shift: quickly create variants without redrawing.
Scene trick: keep related graphics on the same letter in different banks. Bank 1 A might be a stone wall; Bank 2 A the icy version.

4 INK, PAPER and BRIGHT

INK colours the filled pixels. PAPER colours the empty pixels. BRIGHT switches both to their brighter shades. On a real Spectrum one set of these attributes belongs to the whole 8×8 cell—that famous limitation is also where much of the machine's style lives.

The 6×6 Tile Preview

This shows the selected UDG repeated like wallpaper. It reveals unwanted seams and helps with bricks, floors, water, borders and patterns. The INK, PAPER and BRIGHT chosen here are saved with that UDG as its preferred colours.

Screen colours: DEFAULT and CURRENT TILE

DEFAULT

These belong to the whole screen and colour any empty screen cells. Every screen can have a different default INK, PAPER and BRIGHT combination.

CURRENT TILE

These are the INK, PAPER and BRIGHT used for the next tile you paint. Normally they update to the preferred colours saved with each selected UDG.

Turn on Lock colours when several UDGs should share one palette—for example, recolouring an entire second level. Switching UDG or bank will then keep CURRENT TILE unchanged. Turn the lock off and the current UDG's own saved colours return immediately. Painting overrides never alter the UDG's saved preview colours.

5 BASIC DATA without the binary headache

BASIC Export continuously shows the eight decimal bytes for the selected UDG and can produce a listing for the current bank. Copy the selected DATA line for one graphic, or copy the full listing when you want the used part of the bank. Choose the first DATA line number and the increment to match your own program. 

FOR n=0 TO 167
READ a: POKE USR "A"+n,a
NEXT n

The optional POKE loader uses the classic Spectrum arrangement beginning at USR "A". Trailing empty UDG slots are omitted to save memory, while blank slots between used UDGs are kept so later letters still land in the correct place. If you already have your own loader, untick it and use the DATA alone. In 128 BASIC, UDG positions T and U overlap the SPECTRUM and PLAY tokens, so test programs using those letters in the intended BASIC mode.

6 Building complete 32×24 screens

The screen matches the Spectrum's 32 character columns by 24 rows. Choose a UDG from the palette beside it, choose the painting colours, then place tiles. Each painted cell remembers its UDG bank, letter, INK, PAPER and BRIGHT.

  • Previous / Next: move between screens in the project.
  • New Screen: adds a blank screen using the current defaults.
  • Duplicate Screen: makes a complete copy—ideal for menus, room variations or animation states.
  • Delete Screen: removes the current screen; the project always keeps at least one.
  • Zoom: changes editing size only. Fit squeezes all 32 columns into the available width.
  • Undo / Redo: reverses or reapplies the latest screen action as one operation.

On a phone, rotate to landscape for more canvas space. The compact sections keep secondary options folded away, while touch cells use the same friendly tap-to-add, tap-again-to-remove behaviour as the UDG grid.

7 Screen tools

Paint

Draw with the selected UDG. Mouse users right-drag to erase; touch users tap an occupied cell to remove it.

Rectangle Fill

Drag from one corner to the opposite corner to fill an area with the current tile.

Copy Region

Drag around any rectangular area. The copied region stays with the saved project.

Paste Region

Click or tap the cell where the copied rectangle's top-left corner should appear.

Stamp

Repeatedly places the selected UDG. It is useful for scattered objects and deliberate individual placement.

Grid / Clear

Hide the editing grid to inspect the artwork, or clear the whole current screen. Undo is there if your hand slips.

Speedy keyboard selection

When you are not typing in a field, press 1–4 to choose a bank and A–U to choose that UDG letter. Use Ctrl/Cmd+C and V to copy and paste the selected UDG. Use Ctrl/Cmd+Z for Undo and Ctrl/Cmd+Y (or Shift+Z) for Redo.

8 Download TAP: take it to the Spectrum

TAP export is the grand finale. It produces a real tape image containing an auto-running BASIC loader, a compact machine-code screen renderer, every nonblank UDG needed by the project, and all your compressed screens and colours. Blank UDG slots are skipped, so you do not need to fill all 84.

  1. Give the tape a short Spectrum-friendly filename.
  2. Choose Download TAP.
  3. Open the TAP in an emulator, flash-cart workflow, tape utility, or other setup used by your real machine.
  4. Use LOAD "". The supplied loader reserves memory, loads the package and draws screen 1 automatically.

Calling another screen from your BASIC program

LET s=2
POKE 50000,s-1
RANDOMIZE USR 50016

Set s to the screen number you want. BASIC counts from 1, while the renderer counts internally from 0, hence s-1. The package lives at address 50000 and the renderer entry point is 50016. If you combine it with a larger program, reserve the area with CLEAR 49999 and avoid placing other code or variables over it.

The renderer can draw a screen containing UDGs from several banks at once. It also rebuilds the screen's preferred bank in the normal BASIC UDG area, ready for any compatible UDG printing your program performs afterward.

9 Cloud Projects: save, share and play anywhere

The ordinary Save Project button downloads a file to your device. Cloud Projects is the optional online cassette shelf: sign in with Google and you can open your work on another browser or device. Your local browser recovery and downloaded project files keep working whether you use the cloud or not.

Your account and private saves

  1. Choose Cloud Projects at the top of the editor.
  2. Sign in with Google. On your first visit, the site creates an account linked to that verified Google identity; it never receives your Google password.
  3. Enter a cloud project name and choose Save as New.
  4. After opening a saved cloud project, use Update Current or its Save button to replace that same cloud copy.

Save as New is useful for experiments and variations. Update Current keeps working on the selected cloud copy. Projects stay private until you deliberately publish them.

Publishing and sharing

Open one of your cloud projects and choose Publish TAP. The editor first saves the latest project, builds its TAP, and creates two different public addresses:

Project link

Copy Project Link lets somebody view the shared project, download its TAP, or open a copy in their own editor. Opening the shared copy never gives them permission to overwrite yours.

TAP link

Copy TAP Link copies the direct address ending in .tap. Use it with compatible emulators, websites, launchers, or anywhere that needs the tape file itself.

  • Download TAP opens the published tape directly.
  • Try in QAOP sends that TAP address to the QAOP JavaScript Spectrum emulator and starts the quickest part of the development cycle: seeing whether it actually works!
  • Update TAP rebuilds a published TAP after you change the project. Its revisioned link ensures emulators receive the new tape instead of a cached older build.
  • Unpublish removes both public links while keeping the private cloud project.
  • Delete permanently removes the cloud project and any published TAP. Your separately downloaded files are unaffected.
Anyone who has a published link can access it, so share only projects you are happy to make public. There is no public gallery at present: links are unlisted unless you choose to pass them around.

For the site administrator

Administrator accounts see Manage Users. It shows registered users, project counts and storage use. An administrator can disable or re-enable an account and remove an unsuitable project. A disabled user cannot sign in and their published links stop working.

10 The Assembler preview

Choose Assembler in the top toolbar to open the first ZX Spectrum Studio companion tool. It has its own uncluttered workspace, while the UDG Designer remains exactly where you left it in this browser.

  1. Write or paste Z80 source into the editor. The supplied example is ready to assemble.
  2. Choose Assemble to see errors, a memory-address listing and the symbol table.
  3. Use Copy BASIC DATA or Copy hex bytes when another program needs the assembled bytes.
  4. For a TAP that can load and run by itself, keep BASIC loader + code selected and choose Download TAP.

Saving and undoing

Assembler projects have the same safety net as graphics projects. Save Project downloads editable source and export settings as JSON; Load Project restores it. The latest project is also recovered from this browser after a refresh, and the browser warns before leaving with unsaved changes. Undo and Redo buttons—and the familiar keyboard shortcuts—step through source edits.

Assembler projects in the cloud

Cloud assembler projects remain separate from UDG projects but use the same Google account. Save privately, update the current cloud copy, or publish it. Publishing always builds the self-running BASIC-loader TAP, then provides a project link, a direct TAP link and Try in QAOP. A shared project opens as a new editable copy; it never grants permission to overwrite the owner's project.

The Assembly results, Export and Instruction help panels can each be collapsed with their − button. Results stay within a fixed scrolling area, so even a heroic listing will not push Export out of reach. Instruction help begins folded away until you need it.

Graphics and assembler saves are deliberately distinct for now. Passing UDG data into assembly and combining several Studio components into one build will come later, after each tool is solid on its own.

11 Good things to try

  • Build a title screen with a tiled border, logo pieces and menu icons.
  • Design a room in Bank 1, then use Lock colours to create ice, lava or night-time versions.
  • Use matching letters across banks for animation frames or themed replacements.
  • Duplicate a screen before changing it to make doors open, objects disappear, or menus highlight.
  • Copy and paste repeating rooms, status panels, platforms and decorative corners.
  • Turn the grid off regularly. Spectrum graphics often look different when the scaffolding disappears.
  • Test the TAP early in your favourite emulator, then keep iterating. Eight pixels can have a surprising amount of personality.

Have fun, embrace the attribute squares, and make the loading screen you wished you had written in 1984.
